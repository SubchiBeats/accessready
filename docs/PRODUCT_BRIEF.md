# AccessReady Product Brief

## One-line pitch

AccessReady is a Section 508 preflight assistant that helps communications teams catch common accessibility issues before publishing and produce plain-English remediation reports.

## Target users

- Government contractors
- Federal communications teams
- Public health/science communications teams
- Universities and public entities preparing for WCAG workflows
- Nonprofits with public-facing reports and event materials
- Freelancers who deliver websites, PDFs, decks, newsletters, or EPKs

## Core problem

Accessibility workflows often happen too late. Teams export a PDF, publish a webpage, or send an email campaign, then discover issues after approval pressure is already high.

AccessReady shifts remediation earlier by checking source files and content drafts.

## Product principles

1. **Preflight before perfection**: Catch common issues before expert remediation.
2. **Plain English over jargon**: Explain why each issue matters and how to fix it.
3. **Human approval by design**: Never pretend automated checks guarantee compliance.
4. **Evidence matters**: Generate reports that can be saved with QA/QC documentation.
5. **Source files first**: Help teams fix the Word, PowerPoint, Markdown, or HTML source when possible.

## MVP scope

- Upload or scan files
- Identify common issues
- Suggest fixes
- Produce Markdown/JSON reports
- Include manual checklists
- Support CLI, web app, and GitHub Action workflows

## Future differentiators

- Alt text review queue for scientific/public health graphics
- CSV remediation tracker for project managers
- Client-ready evidence packet export
- GitHub pull request comments
- CMS pre-publish checks
- Google Drive/Docs connector concept
- Optional integration with PAC/Acrobat/CommonLook for deeper PDF remediation
