# AccessReady 508 Preflight Checklist

This checklist is not a replacement for your organization's official accessibility review process. It is a practical preflight list for content teams.

## Document basics

- [ ] File has a meaningful title
- [ ] Document language is set
- [ ] Headings are used in logical order
- [ ] Lists are real lists, not manually typed bullets
- [ ] Tables are used for data, not layout
- [ ] Data tables identify header rows and/or columns
- [ ] Links use descriptive text
- [ ] Plain URLs are avoided as link text unless needed

## Images and visual content

- [ ] Meaningful images have accurate alt text
- [ ] Decorative images are marked decorative or use empty alt text
- [ ] Charts include a concise takeaway in alt text or nearby text
- [ ] Complex graphics have a longer description or accessible data table
- [ ] Color is not the only way meaning is communicated
- [ ] Text has sufficient contrast

## Video/audio

- [ ] Videos have captions
- [ ] Audio-only content has transcripts
- [ ] Important visual-only information has audio description or a text alternative
- [ ] Embedded players are keyboard accessible

## PDFs

- [ ] PDF is tagged
- [ ] Reading order is correct
- [ ] Headings, lists, figures, tables, and artifacts are tagged correctly
- [ ] Document title and language are set
- [ ] Bookmarks are included for long documents when appropriate
- [ ] Form fields have labels, names, roles, and values
- [ ] Final PDF passes the organization's required checker and manual review

## Web/HTML

- [ ] Page has a unique title
- [ ] HTML lang attribute is set
- [ ] Images have alt text or are marked decorative
- [ ] Forms have programmatic labels
- [ ] Buttons and icon controls have accessible names
- [ ] Keyboard navigation works
- [ ] Focus indicator is visible
- [ ] Iframes have titles
- [ ] Tables use semantic headers
- [ ] Color contrast meets the applicable threshold
